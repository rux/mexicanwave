Important Change to the Default visitorID Value

AppMeasurement for iOS was updated to no longer use the deprecated UDID value as the default value for visitorID. If you set a custom visitorID in your application (for example, s.visitorID = @12345), then you are unaffected by this change.

If you do not set a custom visitorID, instead of using the UDID as the value for visitorID, a random UUID is generated on initial launch and then stored in a user defaults key. This key is used by AppMeasurement from that point forward. This key is also saved and restored during the standard application backup process.
